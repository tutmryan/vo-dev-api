# Issue your first credential

## Before you begin

:::info[Access]
User will require the `Credential admin` role to create the credential definition and the `Issuer` role to issue the credential.
:::

## TL;DR

If you want to fast-track issuing your first credential, work through the following steps.

- Start in the Credentials area of the Verified Orchestration admin UI and:
  - Add a new credential, confirming the card design, credential types, claim attribute fields and validity details.
  - Ensure you publish your new credential so it can be issued
- From the Credentials area of the admin UI, click through to the details of your new credential:
  - Click the **Issue credential** button to start the issuance
  - Add a new recipient identity and choose to deliver the credential “In person”
  - Enter the credential data to include in the credential
  - If you chose to include a Face check photo in the claim attribute fields, scan the QR code to complete the collection of your Face check selfie.
  - During the Issuance step, scan the QR code using the Verified IDs area of the Microsoft Authenticator app and complete the credential issuance.

### Issue your new credential

Verified Orchestration provide a range of ways to issue credentials to an individual. These include:

1. In person issuance using the admin UI
2. Remote issuance to an individual, verifying their email and mobile number
3. Bulk remote issuance, where remote issuance can be used to distribute thousands of credentials
4. Custom integration via API of any of these methods into a customer app environment.

To get started, we’ll step you through issuing two credentials, one face-to-face and another using the remote issuance capability of the admin UI.

#### ‘In person’ credential issuance

:::info[What is an ‘In person’ issuance?]
An in person issuance is a process that is completed using the Verified Orchestration admin UI, where the operator with access to the admin UI is able to share that same interface with the intended credential recipient. This could be face-to-face or over a video meeting where a screen can be shared.
:::

1. From the **Credentials** area of the admin UI, select the Member credential we created earlier
2. Press the ‘Issue Credential’ button to start the issuance process
   ![Issue credential button](../../static/img/issuance-credential-button.png)
3. From the ‘Recipient’ tab of the issuance flow, enter a human readable ‘Recipient name’ string and select ‘Add “Jane Citizen”’ under the ‘Recipient name’ text input.
   ![Credential recipient tab](../../static/img/credential-recipient.png)
4. Finalise the identity creation by providing an email address for Jane Citizen and press ‘Add’.
   ![Add recipient popup](../../static/img/add-recipient-popup.png)
5. You will be returned to the ‘Recipient’ tab, where you can optionally set an expiry date for the credential and press ‘Next’ to move to ‘Credential Data’
   ![Credential recipient tab](../../static/img/issuance-credential-recipient.png)
6. Provide credential data that satisfies the fields you created earlier and press ‘Next’ to continue.
   ![Credential data](../../static/img/issuance-credential-data.png)
7. Because the credential fields includes a Face check photo, the ‘Photo’ capture step is required. If you have a high quality JPG image of the individual, you can upload this here, or use our ‘Capture photo' feature to have the individual supply their own selfie.
   ![Credential facecheck](../../static/img/issuance-credential-facecheck.png)
   1. To use the ‘Capture photo’ feature, have the recipient scan the QR code and complete a selfie capture and upload from their mobile device.
      ![Capture photo feature](../../static/img/capture-photo-feature.png)
8. When the selfie capture is completed, the issuance flow will automatically move to the ‘Issuance’ tab where a second QR code will be shown. This QR code initiates the credential issuance process into the individuals Microsoft Authenticator app.
   ![Issuance QR code](../../static/img/issuance-qr-code.png)
9. Your unique Verification code will be shown on screen and needs to be entered into Microsoft Authenticator by the credential recipient.
   ![Issuance verification code](../../static/img/issuance-verification-code.png)
   1. The credential recipient can now complete accepting the credential into their Microsoft Authenticator app.
      ![Authenticator adding credential](../../static/img/authenticator-adding-credential.png)
10. You have now successfully issued a credential to the recipient and the issuance flow shows you the successful green check mark.
    ![Issuance done](../../static/img/issuance-done.png)

#### ‘Remote’ credential issuance

:::info[What is a remote issuance?]
Remote credential issuance is a process where the credential offer and it’s acceptance by the intended recipient is undertaken at different times. The identity of the recipient cannot be verified by their attendance, but must be verified by other means as part of the process to accept the credential into the intended recipients Microsoft Authenticator.
:::

To commence a remote credential issuance process, in the ‘Recipient’ tab of the issuance, select the ‘Delivery method’ as **Remote**.

![Remote issuance](../../static/img/issuance-remote.png)

1. 2. 3. Setting up a remote issuance steps you through the same tabs to select the ‘Recipient’, ‘Credential data', and ‘Photo’, although the photo capture process is deferred and occurs as part of the recipient interaction after they receive their credential offer notification.

2. The ‘Issuance’ tab is unique to the remote issuance flow. You can setup the parameters that confirm:
   1. **Issuance expiry:** the period that the offer is valid.
   2. **Notification:** select Email (default) or SMS as the method of delivering a message that the recipient has been offered a credential.
   3. **Verification:** select Email or SMS (default) as the method of providing a one-time-pin that needs to be entered to finalise the credential issuance.

We recommend using different method for the notification and the verification to provide multiple levels of assurance that the intended recipient is receiving the credential.

When you are happy with the contact information of the intended recipient, press ‘Issue’.

![Remote issuance contact information](../../static/img/issuance-contact-information.png)

5. The intended recipient will now receive their notification, which includes a link to finalise the issuance process in Microsoft Authenticator.

![Issuance notification](../../static/img/issuance-notification.png)

6. After clicking the link, the recipient will be directed to the final issuance page where they are asked to enter the **Verification Code**. At the same time, the verification code is delivered to the verification method and target you configured in the issuance.

![ISsuance remote verification code](../../static/img/issuance-remote-verification-code.png)

7. If the credential requires a Face check photo to be captured, the recipient will be asked to capture their selfie from our photo capture portal.
   <br />
   <img
     src={require('../../static/img/issuance-remote-facecheck-photo.png').default}
     alt="Issuance facecheck photo"
     style={{ width: 372 }}
   />
8. The final step in the issuance flow is to present the QR code to issue the credential to the recipient’s Microsoft Authenticator app.
   <br />
   <img src={require('../../static/img/issuance-remote-qr-code.png').default} alt="Issuance QR code" style={{ width: 372 }} />
9. The credential recipient can now complete accepting the credential into their Microsoft Authenticator app as they would with any other issuance.
   ![Authenticator adding credential](../../static/img/authenticator-adding-credential.png)

### Confirm the credential issuance

Use the admin UI and visit the **Issuances** page. This page shows you the history of ‘In person' issuances that have been completed. This include:

- ‘In person’ issuances via the admin UI
- ‘In person’ issuances completed via other apps that use the issuance API.

![List of issuances](../../static/img/issuances-list.png)

If you click through to the issuance ‘Details’, you will be presented a page that includes:

- The Credential name
- The recipient identity information and how the credential was issued
- Which user issued the credential

:::note
If the credential is issued using an integrated application, the ‘Issued by’ field will show the name of the integrated application.
:::

- When the credential was issued and when it will expire
- The current status of the credential
- Whether the credential includes a Face check photo
