# Onboarding an app

For apps to be able to integrate with Verified Orchestartion platform, they will need to be granted access to Verified Orchestration enterprise application roles.

A tenant administrator can help onboarding an app in Microsoft Azure Portal [[doc](https://learn.microsoft.com/en-us/azure/healthcare-apis/register-application)].

## Access models

Primary access to Verified Orchestration platform is via a client secret. A second access model caters for apps which do not have the ability to securely store a secret. Limited access tokens are used in this case.

### Limited access token model

This model is best suited for the client apps which do not have the ability to securely store a secret such as Single Page Applications (SPA), mobile apps, etc. The following application permissions from Verified Orchestration API are applicable to this access model.

- **VerifiableCredential.AcquireLimitedAccessToken.AnonymousPresentations**<br/>
  Acquire limited access token: anonymous presentations
- **VerifiableCredential.AcquireLimitedAccessToken.Issue**<br/>
  Acquire limited access token: issue
- **VerifiableCredential.AcquireLimitedAccessToken.ListContracts**<br/>
  Acquire limited access token: list contracts
- **VerifiableCredential.AcquireLimitedAccessToken.Present**<br/>
  Acquire limited access token: present

For more information on how the limited access token works, refer to the [Limited access tokens guide](./limited-access-tokens/).

### Conventional access token model

This model is also available for the apps which can securely store a secret such as APIs, automation workflows, etc. The following application permissions from Verified Orchestration API are applicable to this access model.

- **VerifiableCredential.Issue**<br/>
  Credential issuance
- **VerifiableCredential.Present**<br/>
  Credential presentation
- **VerifiableCredential.RequestApproval**<br/>
  Create approval requests and read actioned approval data

These permissions can be granted to the App registration depending on the requirements of the consumer app.

## Instance configuration

Refer to the [Instance configuration](./instance-configuration/) guide for the following configuration settings:

### Application label

When you onboard an application, you can supply a user-friendly label, helping users to understand which apps are issuing or requesting presentation of credentials.

### CORS origin

If you are using a web application to access the Verified Orchestration API, the host address must be configured as a CORS origin.
