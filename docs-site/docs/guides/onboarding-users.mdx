# Onboarding a user

Individuals who would like to access the <AdminUrl path='/'>Admin UI</AdminUrl> or <ApiUrl path='/'>GraphQL API</ApiUrl> will need to be granted access via Verified Orchestration enterprise application user roles.

:::tip
The name of the Verified Orchestration platform application in Azure AD is: **Verified Orchestration (<Instance />)**
:::

We recommend creating Microsoft Entra groups mapped to each Verified Orchestration user role. Users can then be added to one or more groups, according to the tasks they would like to carry out on the platform.

- [Reader](#reader)
- [Issuer](#issuer)
- [Credential admin](#credential-admin)
- [Partner admin](#partner-admin)
- [Approval request admin](#approval-request-admin)

A tenant administrator can set up the the group to role mapping and add/remove users from the groups.

## Reader

App role value: `VerifiableCredential.Reader`

Permissions:

- view templates
- view credentials
- view issuances
- view presentations
- view identities

## Issuer

App role value: `VerifiableCredential.Issuer`

Permissions:

- all the permissions of [Reader](#reader) role, and
- create identity
- update identity
- issue credential

## Credential admin

App role value: `VerifiableCredential.CredentialAdmin`

Permissions:

- all the permissions of [Reader](#reader) role, and
- create template
- edit template
- delete template
- create contract
- edit contract
- delete contract
- publish contract
- deprecate contract
- create identity
- update identity
- revoke issuances

## Partner admin

App role value: `VerifiableCredential.PartnerAdmin`

Permissions:

- all the permissions of [Reader](#reader) role, and
- find authorities / issuers in verifiable credentials network
- find contracts / credentials in verifiable credentials network
- add partner
- edit partner

## Approval request admin

App role value: `VerifiableCredential.ApprovalRequestAdmin`

Permissions:

- view approval requests
- view actioned approval data
