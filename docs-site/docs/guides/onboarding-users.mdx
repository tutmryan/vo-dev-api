# Onboarding a user

The individuals who would like to access <AdminUrl path='/'>Admin UI</AdminUrl> or <ApiUrl path='/'>GraphQL API</ApiUrl> will need to be onboarded to Microsoft Entra ID tenant of Verified Orchestration platform and added to one or more of the following Microsoft Entra ID groups depending on the tasks they would like to carry out on the platform.

- [Reader](#reader)
- [Issuer](#issuer)
- [Credential admin](#credential-admin)
- [Partner admin](#partner-admin)

Each of the above Microsoft Entra ID groups is mapped to the equivalent App role of the Verified Orchestration API App registration.

An administrator of Verified Orchestration platform would be able to help with onboarding an individual and adding him/her to the groups in Microsoft Azure Portal [[doc](https://learn.microsoft.com/en-us/azure/active-directory/fundamentals/how-to-create-delete-users)].

## Reader

Microsoft Entra ID Group: `Verified Orchestration Reader`

App role value: `VerifiableCredential.Reader`

Permissions:

- view templates
- view credentials
- view issuances
- view presentations
- view identities

## Issuer

Microsoft Entra ID Group: `Verified Orchestration Issuer`

App role value: `VerifiableCredential.Issuer`

Permissions:

- all the permissions of [Reader](#reader) role, and
- create identity
- update identity
- issue credential

## Credential admin

Microsoft Entra ID Group: `Verified Orchestration Credential Admin`

App role value: `VerifiableCredential.CredentialAdmin`

Permissions:

- all the permissions of [Reader](#reader) role, and
- create template
- edit template
- delete template
- create contract
- edit contract
- delete contract
- publish contract
- deprecate contract
- create identity
- update identity
- revoke issuances

## Partner admin

Microsoft Entra ID Group: `Verified Orchestration Partner Admin`

App role value: `VerifiableCredential.PartnerAdmin`

Permissions:

- all the permissions of [Reader](#reader) role, and
- find authorities / issuers in verifiable credentials network
- find contracts / credentials in verifiable credentials network
- add partner
- edit partner
