# Onboarding a user

Individuals who would like to access the <AdminUrl path='/'>Admin UI</AdminUrl> or <ApiUrl path='/'>GraphQL API</ApiUrl> will need to be granted Verified Orchestration enterprise application user-role access. We suggest creating Microsoft Entra groups mapped to each Verified Orchestration  user role. Users can then be added to one or more groups depending on the tasks they would like to carry out on the platform.

- [Reader](#reader)
- [Issuer](#issuer)
- [Credential admin](#credential-admin)
- [Partner admin](#partner-admin)
- [Approval request admin](#approval-request-admin)

A tenant administrator can set up the the group to role mapping and add/remove users from the groups.

## Reader

Microsoft Entra ID Group: `Verified Orchestration Reader`

App role value: `VerifiableCredential.Reader`

Permissions:

- view templates
- view credentials
- view issuances
- view presentations
- view identities

## Issuer

Microsoft Entra ID Group: `Verified Orchestration Issuer`

App role value: `VerifiableCredential.Issuer`

Permissions:

- all the permissions of [Reader](#reader) role, and
- create identity
- update identity
- issue credential

## Credential admin

Microsoft Entra ID Group: `Verified Orchestration Credential Admin`

App role value: `VerifiableCredential.CredentialAdmin`

Permissions:

- all the permissions of [Reader](#reader) role, and
- create template
- edit template
- delete template
- create contract
- edit contract
- delete contract
- publish contract
- deprecate contract
- create identity
- update identity
- revoke issuances

## Partner admin

Microsoft Entra ID Group: `Verified Orchestration Partner Admin`

App role value: `VerifiableCredential.PartnerAdmin`

Permissions:

- all the permissions of [Reader](#reader) role, and
- find authorities / issuers in verifiable credentials network
- find contracts / credentials in verifiable credentials network
- add partner
- edit partner

## Approval request admin

Microsoft Entra ID Group: `Verified Orchestration Approval Request Admin`

App role value: `VerifiableCredential.ApprovalRequestAdmin`

Permissions:

- view approval requests
- view actioned approval data
