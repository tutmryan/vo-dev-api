# Application integration

The Verified Orchestration platform is based on a GraphQL API which supports integration with custom applications.

We provide info and examples for the following development stacks:

- [JavaScript using client library](javascript-client)
- [React + Apollo Client](react-apollo)
- [iOS Swift + Apollo Client](ios-apollo)
- [Android Kotlin + Apollo Client](android-apollo)

You may wish to use a completely different development stack. The guides and examples should still be useful as a reference. Refer to the [GraphQL API](#graphql-api) section for more info on integration to the API via whatever development stack you use.

## GraphQL API

The Verified Orchestration platform is based on a GraphQL API. All data and operations are available through the API, which presents a strongly typed, documented and discoverable schema.

### Apollo Sandbox UI

Apollo Sandbox is a GraphQL IDE that allows you to explore the Verified Orchestration API. It is useful for exploring the schema, documentation, building and running queries.

You will need to be [onboarded as a platform user](../onboarding-users.mdx) to access Apollo Sandbox and the API.

#### Basic Apollo Sandbox usage

- Use the Documentation tab in the Explorer view to navigate the schema and select operations, types and fields.
- Use the Variables tab to define operation variables.
- The Traces view can help you understand the performance of your query.
- Use the Share button to generate a link to an operation definition.
- Export data as JSON or CSV.

#### Subscriptions in Apollo Sandbox

Query and Mutation operations work *out of the box* in Apollo Sandbox using cookie authentication (see the *Include cookies* setting in *Connection settings*).

Subscription connections do not include cookies, will need to provide an Authorization header for the subscription operation.

1. Copy an Authorization header value from the <AdminUrl>Admin website</AdminUrl> using your browser dev tools
1. Paste the Authorization header into the **Headers** section in Apollo Sandbox

You can now subscribe to events and see event data. An access token normally lasts for around 50 minutes depending on your tenant configuration.
