# Authentication and authorization

As described in the [onboarding guide](../onboarding-apps.mdx), different types of applications will use complementary auth models:

- [Conventional access token model](/docs/guides/onboarding-apps#conventional-access-token-model) for secure backend APIs
- [Limited access token model](/docs/guides/onboarding-apps#limited-access-token-model) for client applications

Code samples for both models are shown in the [Secure backend API for limited access tokens](/docs/guides/limited-access-tokens/secure-backend-api) including:

- [Get a backend access token](/docs/guides/limited-access-tokens/secure-backend-api#get-access-token-to-call-verification-orchestration-platform)
- [Get a client access token](/docs/guides/limited-access-tokens/secure-backend-api#get-limited-access-token-from-the-verified-orchestration-platform)
