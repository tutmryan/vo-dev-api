As well as providing back-end infrastructure improvements, this release introduces retention of verifiable presentation receipts.

<!-- truncate -->

## 🚀 What’s new
### New features
- Persist presentation receipt (partial, id_token redacted)
- Introduce retention of a wallet object and identifier
- Added support for confidential clients (client secret) in the OIDC provider
- Introduce claim mapping in the OIDC provider response to the client

### Enhancements
- Support cryptographic verification of presentation receipts in the Composer
- Include presentation constraints in the Composer UI
- Display the partner DID when searching for partners

## 🛠️ Notable fixes
- Adjust OTP throttling to address verification challenges not completing
- Resolve problem switching between desktop and mobile to capture FaceCheck photos
